# Quickstart

This quickstart is for someone discovering OpenHuman Online from GitHub, a search result, a comparison article, or a product directory.

## 1. Open The Hosted SaaS

Start here:

- https://openhuman.online/?utm_source=github&utm_medium=documentation&utm_campaign=openhuman_online_docs&utm_content=quickstart_home

The first visit should answer three questions:

- Does the first screen make the hosted personal AI memory workspace workflow obvious?
- Is there a usable preview or concrete product surface before pricing?
- Is the Team annual path easy to find?

## 2. Learn The Workflow

1. Start with a small memory pilot and a clear owner.
2. Choose the first source set: notes, meetings, GitHub, Gmail, Calendar, or documents.
3. Generate a Memory Tree convention, meeting brief, and follow-up template.
4. Review privacy boundaries before adding higher-context chat or repository history.

## 3. Compare Against Your Current Process

Use this simple evaluation prompt internally:

```text
Where does OpenHuman Online reduce repeated manual work?
What private data would it touch?
Who approves the first rollout?
What evidence do we need before paying for the default plan?
```

## 4. Review Pricing

Open the tracked pricing link:

- https://openhuman.online/pricing/?utm_source=github&utm_medium=documentation&utm_campaign=openhuman_online_docs&utm_content=quickstart_pricing

The default buyer path is Team annual. If a different plan is more appropriate, document why before checkout.
